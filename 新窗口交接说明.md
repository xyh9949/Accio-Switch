# Accio Switch 新窗口交接说明

## 项目目标

Accio Switch 是一个 Windows 本地桥接工具，用于让 Accio Work 使用第三方 OpenAI 兼容接口。

当前原则：

- 普通聊天、技能调用、插件调用走第三方聊天模型。
- 生图和改图可配置独立的第三方模型/API。
- Custom 模式下，模型请求不得回退到 Accio 官方模型接口。
- Accio 的登录、账户、插件管理等非模型 API 仍透明转发，以保证客户端正常运行。

## 已解决的重要问题

### 1. 普通聊天返回空响应

Accio SDK 需要 SSE 格式响应。桥接目前使用：

```text
Content-Type: text/event-stream
data: {...}
```

### 2. 技能和插件参数变成空对象

Accio proto 工具调用要求：

```json
{
  "functionCall": {
    "id": "call_xxx",
    "name": "skill",
    "argsJson": "{\"action\":\"read\",\"skill_id\":\"pdf\"}"
  }
}
```

不能只发送 `args`。否则 Accio SDK 会保留工具名和 ID，但将参数解析为 `{}`。

当前桥接还支持：

- 对象形式的工具参数
- JSON 字符串参数
- SSE 参数分片
- OpenAI Responses API 的 `function_call`
- 缺少必填工具参数时聚焦重试一次

### 3. 生图请求只返回文字

Accio 的 `image_generate` 和 `image_edit` 会发送：

```json
{
  "generationConfig": {
    "responseModalities": ["TEXT", "IMAGE"]
  }
}
```

0.2.0 起支持独立生图路由：

- `Chat Completions multimodal`
- OpenAI Images API：
  - `/images/generations`
  - `/images/edits`

生图可独立设置：

- Base URL
- API Key
- 模型
- 协议

图片响应支持：

- base64
- Data URL
- HTTP/HTTPS URL
- JSON
- SSE

图片最终转换成 Accio 所需格式：

```json
{
  "inlineData": {
    "mimeType": "image/png",
    "data": "base64..."
  }
}
```

### 4. 国际站生意助手出现 HTML 502

已确认国际站插件和 Alibaba MCP 工具可以加载。

失败原因是：

```text
accio-mcp-cli search alibaba
```

返回约 114,000 字符，完整放入下一轮模型上下文后，第三方网关返回 Cloudflare 风格 HTML 502 页面。

0.2.1 已加入：

- 单条工具结果最大约 32 KB
- 保留结果开头约 67% 和结尾约 33%
- 中间加入压缩说明
- 第三方 HTTP 5xx 自动重试一次
- HTML 错误页显示明确错误，不再显示 `Unexpected token '<'`
- 日志记录每次上游请求的 `requestChars`

压缩仅影响发送给模型的上下文，不修改 Accio 本地保存的原始工具输出。

## 使用和测试注意事项

1. 启动新版前，必须完全退出 Accio Work 和旧版 Accio Switch。
2. 从 Accio Switch 点击 `Launch Accio`，确保环境变量在 Accio 启动时生效。
3. 聊天接口测试成功不代表生图接口可用，生图路由需要单独配置和测试。
4. `image-prompt-guide` 未安装只会影响提示词工作流，不是图片响应缺失的根本原因。
5. 国际站助手失败时，先查看 `bridge.log` 中：
   - `requestChars`
   - HTTP 状态码
   - 是否返回 HTML
   - 工具结果是否过大
6. 不要用 Accio 官方模型回退来掩盖第三方接口错误。
